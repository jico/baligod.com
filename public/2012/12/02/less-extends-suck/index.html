The funny thing that’s not so funny is, we use periods to include LESS mixins as well, which can make our LESS code pretty convoluted. So what happens when we have a class and a mixin called foo? It turns out that both the foo mixin is included and the .foo class extended. Obviously this conflict will only occur when extending classes, but that’s likely 99% of the time. Granted we should probably avoid situations like this, but still, yikes.

But imagine a large project where we want to modularize our CSS for easy maintainability and updates. Extending large classes would keep things very clean, but the output of these extends may add unnecessary weight to your CSS files.
